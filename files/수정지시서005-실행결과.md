# 수정지시서 005 실행결과

## 1. List 계열 메서드에 QueryOpts 전달 — 완료

`buildTemplateData()`에 `st`, `funcName` 파라미터 추가. get 시퀀스에서 해당 operation에 `HasQueryOpts()`이면 `opts` 인자를 자동 추가.

`GenerateFunc()` 내에서 QueryOpts 있으면 함수 본문 상단에 구성 코드 생성:

```go
opts := QueryOpts{}
```

빈 struct 전달로 단순화 (지시서 권장). 실제 `r.URL.Query()` 바인딩은 후속 작업.

생성 결과:
```go
// list_my_reservations.go
opts := QueryOpts{}
reservations, total, err := reservationModel.ListByUserID(currentUser.UserID, opts)
```

## 2. 파생 파라미터/리터럴 인터페이스 포함 — 완료

`resolveParamName()` 수정 — dot notation 결합:
- `user.ID` → `userID`
- `course.Price` → `coursePrice`

`resolveParamType()` 수정 — dot notation DDL 조회:
- `user.ID` → `users` 테이블 `id` 컬럼 → `int64`

리터럴 파라미터 — DDL 역매핑 (`resolveLiteralParamName()` 신규):
- 모델 테이블에서 이미 사용된 컬럼과 자동 생성 컬럼(id, created_at 등)을 제외
- 남은 string 타입 컬럼을 리터럴에 매핑
- `"cancelled"` → `reservations` 테이블에서 → `status`

생성 결과:
```go
// 수정 전
type SessionModel interface {
    Create() (*Token, error)                         // user.ID 누락
}
// 수정 후
type SessionModel interface {
    Create(userID int64) (*Token, error)              // dot notation 포함
}

// 수정 전
UpdateStatus(reservationID int64) error              // "cancelled" 누락
// 수정 후
UpdateStatus(reservationID int64, status string) error  // 리터럴 역매핑
```

## 3. List 계열 반환값 3-tuple — 완료

`templateData`에 `HasTotal` 필드 추가. get 시퀀스 + `[]` 결과 타입 + `HasQueryOpts` → `HasTotal = true`.

```
{{.Result.Var}}, {{if .HasTotal}}total, {{end}}err := {{.ModelVar}}.{{.ModelMethod}}({{.ParamArgs}})
```

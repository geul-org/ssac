# 수정지시서 006: service/ 직접 .go 파일 금지 — 도메인 서브 폴더 강제

## 발견 경위

fullend dummy-gigbridge에서 `service/*.go` (flat 구조)로 SSaC를 작성했더니 ssac validate 통과 후 코드젠에서 flat mode가 적용되어 `CurrentUser`가 `model/auth.go`에 생성되지 않음. domain mode가 필수 정책인데 ssac이 강제하지 않음.

## 현재 동작

`ssac parser.ParseDir()` — `service/login.go` 같이 서브 폴더 없는 파일도 `Domain: ""` 으로 정상 파싱됨.

## 기대 동작

`service/` 직접에 `.go` 파일이 있으면 ERROR:
```
service/login.go — service/ 직접에 SSaC 파일을 둘 수 없습니다. 도메인 서브 폴더를 사용하세요 (예: service/auth/login.go)
```

## 수정 위치

### validator (또는 parser)

`ParseDir()` 결과에서 `Domain == ""` 인 함수가 있으면 ERROR 반환.

또는 `ParseDir()` 단계에서 `filepath.Dir(rel) == "."` 인 파일을 발견하면 에러.

## 매뉴얼 수정

ssac 매뉴얼에도 `service/<domain>/*.go` 서브 폴더 필수 정책을 명시해야 함.

## 검증 방법

```bash
# flat 구조 → ERROR
echo 'package service
func Login() {}' > /tmp/test-ssac/service/login.go
ssac validate /tmp/test-ssac/service  # → ERROR

# 서브 폴더 구조 → OK
mkdir -p /tmp/test-ssac/service/auth
mv /tmp/test-ssac/service/login.go /tmp/test-ssac/service/auth/
ssac validate /tmp/test-ssac/service  # → OK
```

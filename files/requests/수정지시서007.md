# 수정지시서 007: ssac-gen 미사용 import 제거

## 발신
fullend (BUG012)

## 심각도
HIGH — 컴파일 에러

## 현상

ssac-gen이 서비스 파일 생성 시 `"database/sql"`, `"strconv"` 등을 일괄 import하지만, 실제 사용하지 않는 파일에서도 포함되어 `go build` 시 `imported and not used` 에러로 빌드 실패.

```
internal/service/auth/register.go:4:2: "database/sql" imported and not used
internal/service/project/create_project.go:5:2: "database/sql" imported and not used
internal/service/project/list_projects.go:7:2: "strconv" imported and not used
... (10개 파일)
```

## 원인

서비스 파일 생성 시 import를 사용 여부와 무관하게 고정 목록으로 삽입.

- `"database/sql"` — `h.DB.BeginTx` 등 트랜잭션 코드가 있는 경우만 필요하지만 모든 서비스 파일에 포함
- `"strconv"` — `strconv.Atoi` 등 수동 파싱 코드가 제거된 후에도 import 잔존

## 기대 동작

import를 사용 여부에 따라 조건부 생성:

- `"database/sql"` → 생성된 코드에 `sql.` 참조가 있을 때만
- `"strconv"` → 생성된 코드에 `strconv.` 참조가 있을 때만
- 기타 동일 패턴 적용

## 수정 제안

방법 A (권장): 코드 생성 후 import 블록을 생성하기 전에, 생성된 함수 본문을 스캔하여 실제 사용하는 패키지만 import에 포함.

방법 B: 코드 생성 완료 후 `goimports` 또는 동등 로직으로 미사용 import 정리 후처리.

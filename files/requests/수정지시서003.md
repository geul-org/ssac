# 수정지시서 003: 쓰기 시퀀스 자동 트랜잭션 래핑

## 선행 조건

수정지시서001 (파일 분리 리팩토링) 완료 후 진행.

## 배경

SSaC gen이 트랜잭션 코드를 생성하지 않음. `@put` + `@post`가 동시에 있는 핸들러에서 첫 번째 쓰기는 성공하고 두 번째가 실패하면 데이터 불일치 발생.

원래 설계에 `@transaction` 어노테이션이 있었으나 구현 누락됨. 재검토 결과, 어노테이션 없이 **쓰기 시퀀스가 있으면 무조건 트랜잭션**이 올바른 방침.

## 규칙

- 함수에 `@post`, `@put`, `@delete` 시퀀스가 **하나라도 있으면** 트랜잭션 래핑
- `@get`만 있는 함수는 트랜잭션 없음
- 어노테이션 없음, 옵션 없음, 예외 없음

## 생성 코드 변경

### 현재 (트랜잭션 없음)

```go
func (h *Handler) AcceptProposal(c *gin.Context) {
    // ...
    err := h.ProposalModel.UpdateStatus(id, "accepted")
    if err != nil {
        c.JSON(500, gin.H{"error": "update failed"})
        return
    }
    tx, err := h.TransactionModel.Create(gigID, freelancerID, amount)
    if err != nil {
        // ProposalModel.UpdateStatus는 이미 커밋됨 — 불일치
        c.JSON(500, gin.H{"error": "create failed"})
        return
    }
    c.JSON(200, gin.H{"proposal": proposal})
}
```

### 기대 결과 (자동 트랜잭션)

```go
func (h *Handler) AcceptProposal(c *gin.Context) {
    // ...
    tx, err := h.DB.BeginTx(c.Request.Context(), nil)
    if err != nil {
        c.JSON(500, gin.H{"error": "transaction failed"})
        return
    }
    defer tx.Rollback()

    err = h.ProposalModel.WithTx(tx).UpdateStatus(id, "accepted")
    if err != nil {
        c.JSON(500, gin.H{"error": "update failed"})
        return
    }
    transaction, err := h.TransactionModel.WithTx(tx).Create(gigID, freelancerID, amount)
    if err != nil {
        // 자동 롤백 — ProposalModel.UpdateStatus도 취소됨
        c.JSON(500, gin.H{"error": "create failed"})
        return
    }

    if err := tx.Commit(); err != nil {
        c.JSON(500, gin.H{"error": "commit failed"})
        return
    }
    c.JSON(200, gin.H{"proposal": proposal})
}
```

## 구현 요구사항

### 1. 쓰기 감지

코드젠 시 ServiceFunc의 시퀀스 목록에서 `@post`, `@put`, `@delete` 존재 여부 확인.

### 2. Handler에 DB 주입

트랜잭션 생성을 위해 Handler struct에 `*sql.DB` 필드 필요.

```go
type Handler struct {
    DB             *sql.DB
    ProposalModel  model.ProposalModel
    // ...
}
```

### 3. Model에 WithTx 메서드

모델 interface에 `WithTx(*sql.Tx)` 메서드 추가. 트랜잭션 컨텍스트에서 동작하는 모델 인스턴스 반환.

```go
type ProposalModel interface {
    WithTx(tx *sql.Tx) ProposalModel
    Create(...) (*Proposal, error)
    UpdateStatus(id int64, status string) error
    // ...
}
```

구현:

```go
type proposalModelImpl struct {
    db interface{ QueryRowContext(...) *sql.Row }  // *sql.DB 또는 *sql.Tx
}

func (m *proposalModelImpl) WithTx(tx *sql.Tx) ProposalModel {
    return &proposalModelImpl{db: tx}
}
```

### 4. 읽기 전용 함수는 변경 없음

`@get`만 있는 함수는 기존과 동일하게 생성. 트랜잭션 코드 없음.

## 대상 파일

- SSaC 코드젠 — 쓰기 시퀀스 감지 + `BeginTx`/`Commit`/`defer Rollback` 생성
- SSaC 모델 코드젠 — `WithTx` interface 메서드 + 구현 생성
- SSaC 핸들러 코드젠 — 쓰기 시 `h.Model.WithTx(tx).Method(...)` 호출 생성

## 검증

1. 쓰기 2개 이상 함수 → `BeginTx` + `defer Rollback` + `Commit` 코드 존재
2. 쓰기 1개 함수 → 동일하게 트랜잭션 래핑 (단일 쓰기도 일관성)
3. 읽기 전용 함수 → 트랜잭션 코드 없음
4. 모델 interface에 `WithTx` 메서드 존재
5. 기존 테스트 전체 통과

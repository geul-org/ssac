# 수정지시서 020: DDL Index UNIQUE 제약 + PRIMARY KEY 추적

## 요청자

fullend — Phase015 cursor pagination 교차 검증

## 배경

fullend에서 cursor pagination 구현 중, `x-pagination: cursor` + `x-sort` 동시 선언 시 sort 컬럼이 DDL UNIQUE인지 검증해야 합니다. 중복값이 있는 컬럼을 cursor로 쓰면 페이지네이션이 깨지기 때문입니다.

현재 `Index` struct에 UNIQUE 여부 정보가 없고, PRIMARY KEY도 저장되지 않습니다.

## 변경 요청

### 1. `validator/symbol.go` — `Index` struct에 `IsUnique` 추가

```go
// 현재 (line 65-68)
type Index struct {
    Name    string
    Columns []string
}

// 요청
type Index struct {
    Name     string
    Columns  []string
    IsUnique bool // UNIQUE INDEX 또는 UNIQUE 제약
}
```

### 2. `validator/symbol.go` — `DDLTable`에 `PrimaryKey` 추가

```go
// 현재 (line 50-55)
type DDLTable struct {
    Columns     map[string]string
    ColumnOrder []string
    ForeignKeys []ForeignKey
    Indexes     []Index
}

// 요청
type DDLTable struct {
    Columns     map[string]string
    ColumnOrder []string
    ForeignKeys []ForeignKey
    Indexes     []Index
    PrimaryKey  []string // PK 컬럼명 목록 (보통 ["id"])
}
```

### 3. `validator/symbol.go` — `parseCreateIndex` 수정 (line 918-958)

`CREATE UNIQUE INDEX` 파싱 시 `IsUnique: true` 설정:

```go
// 현재 (line 955)
t.Indexes = append(t.Indexes, Index{Name: idxName, Columns: cols})

// 요청
t.Indexes = append(t.Indexes, Index{Name: idxName, Columns: cols, IsUnique: isUnique})
```

`isUnique`는 line 801 부근에서 `CREATE UNIQUE INDEX` 감지 시 true.

### 4. `validator/symbol.go` — `parseDDLTables` 수정

PRIMARY KEY 파싱 추가. 현재 line 841-844에서 `PRIMARY KEY`를 스킵하고 있음:

```go
// 현재: PRIMARY KEY를 무시
if upperFirst == "PRIMARY" { continue }

// 요청: PRIMARY KEY 컬럼 추출
if upperFirst == "PRIMARY" {
    // "PRIMARY KEY (id)" 또는 "PRIMARY KEY (col1, col2)" 파싱
    t.PrimaryKey = extractPKColumns(trimmed)
    continue
}
```

인라인 PK (`id BIGSERIAL PRIMARY KEY`)도 추적:
```go
if strings.Contains(strings.ToUpper(trimmed), "PRIMARY KEY") {
    t.PrimaryKey = append(t.PrimaryKey, colName)
}
```

### 5. UNIQUE 제약 (인라인) 파싱

```sql
email VARCHAR(255) NOT NULL UNIQUE
```

위 형태도 `IsUnique` 단일 컬럼 인덱스로 추가:
```go
if strings.Contains(strings.ToUpper(trimmed), "UNIQUE") && upperFirst != "UNIQUE" {
    t.Indexes = append(t.Indexes, Index{
        Name: colName + "_unique", Columns: []string{colName}, IsUnique: true,
    })
}
```

## 사용처

fullend `internal/crosscheck/openapi_ddl.go`에서:
```go
func isUniqueColumn(col, tableName string, st *SymbolTable) bool {
    table := st.DDLTables[tableName]
    for _, pk := range table.PrimaryKey {
        if pk == col { return true }
    }
    for _, idx := range table.Indexes {
        if idx.IsUnique && len(idx.Columns) == 1 && idx.Columns[0] == col {
            return true
        }
    }
    return false
}
```

## 영향 범위

- `Index` struct 필드 추가 — 기존 코드에서 `IsUnique`를 안 읽으면 영향 없음 (zero value = false)
- `DDLTable.PrimaryKey` 추가 — 기존 코드에서 안 읽으면 영향 없음 (zero value = nil)
- 기존 테스트: 빌드 에러 없음 (필드 추가만)

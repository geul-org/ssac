# 수정지시서 016: 패키지 접두사 @model의 Go interface 교차 검증

## 배경

fullend에서 @model 판단 규칙이 패키지 접두사 기반으로 확정됨:

- **접두사 없음** (`@model Gig.FindByID(...)`) → DDL 테이블 기반 DB 모델
- **접두사 있음** (`@model session.Session.Get(...)`) → 해당 패키지의 Go interface 기반 모델

접두사가 있는 @model의 경우, 해당 패키지 경로에서 Go interface를 파싱하여 SSaC @model 호출과 교차 검증해야 함.
이 검증은 SSaC validate의 업무 범위에 해당 (SSaC가 이미 @model 시퀀스를 파싱하고 검증하므로).

## 요청 사항

### 1. 패키지 접두사 @model 파싱

SSaC 서비스 코드에서 패키지 접두사가 포함된 @model을 파싱할 수 있어야 함:

```go
// 기존 (접두사 없음 — DDL 모델)
// @get User user = User.FindByID({ID: request.ID})

// 신규 (접두사 있음 — 패키지 모델)
// @get Session session = session.Session.Get({token: request.Token})
// @post CacheResult result = cache.Cache.Set({key: "gig:" + gig.ID, value: gig, TTL: 300})
// @post FileResult result = file.File.Upload({key: path, body: request.File})
// @post HoldResponse result = escrow.Escrow.Hold({GigID: gig.ID, Amount: gig.Budget})
```

파싱 결과에 패키지명이 포함되어야 함:
- `session.Session.Get` → 패키지: `session`, 모델: `Session`, 메서드: `Get`
- `User.FindByID` → 패키지: (없음), 모델: `User`, 메서드: `FindByID`

### 2. Go interface 파싱 + 교차 검증

패키지 접두사가 있는 @model에 대해:

1. 해당 패키지 경로에서 Go interface 파일을 탐색
2. interface에 정의된 메서드명, 파라미터, 리턴 타입 추출
3. SSaC @model 호출의 메서드명, 파라미터가 interface와 일치하는지 검증

**검증 규칙:**
- 메서드가 interface에 없음 → ERROR
- 파라미터 이름이 interface 메서드 시그니처와 불일치 → ERROR (가능한 경우)
- 패키지 경로에 interface 없음 → WARNING ("검증 불가 — interface 미제공")
- 패키지 경로 자체가 없음 → ERROR

### 3. 패키지 경로 탐색 규칙

패키지명으로부터 실제 경로를 찾는 방법:
- SSaC 서비스 파일 상단의 import 선언 활용 (기존 @call func과 동일 패턴)
- 또는 fullend가 패키지 경로 목록을 ssac validate에 전달

이 부분은 fullend와 SSaC 간 인터페이스 협의가 필요할 수 있음.

## 예시

```go
// pkg/session/session.go
type SessionModel interface {
    Set(ctx context.Context, key string, value any, ttl time.Duration) error
    Get(ctx context.Context, key string) (string, error)
    Delete(ctx context.Context, key string) error
}
```

SSaC 서비스:
```go
// @get Session session = session.Session.Get({token: request.Token})
```

ssac validate:
- `session` 패키지에서 `SessionModel` interface 탐색
- `Get` 메서드 존재 확인 → OK
- 파라미터 `token` 대응 확인

## 모델 정책

1. **fullend/pkg 모델도 특별 취급 없음** — 사용자 커스텀과 동일한 검증 경로. fullend/pkg/session이든 사용자가 만든 mysession이든 같은 방식으로 interface를 파싱하고 검증.
2. **범용 모델(Session, Cache, File, External)은 fullend/pkg가 기본 제공**, 사용은 선택. 사용자가 자기 패키지로 대체 가능.
3. **특수 모델은 사용자가 interface + 구현체 직접 작성** 정책. interface만 제공하면 validate 혜택을 받을 수 있음.
4. **validate 에러 시 interface의 사용 가능 메서드 목록 안내** (자기 교정 루프). 예: `[ERROR] session.Session — 메서드 "Gett" 없음. 사용 가능: Set, Get, Delete`

# 수정지시서 005: @call Request struct named field 생성 + trailing dot 수정

## 발견 경위

fullend dummy-gigbridge 프로젝트 코드젠 중 ssac-gen이 `@call` 시퀀스에서 잘못된 Go 코드를 생성함.

## 재현 SSaC

```go
// @call string hashedPassword = auth.HashPassword(request.Password)
// @post User user = User.Create(request.Email, hashedPassword, request.Role, request.Name)
func Register() {}
```

## 현재 생성 코드 (잘못됨)

```go
hashedPassword, err := auth.HashPassword(auth.HashPasswordRequest{ password })
// → 컴파일 에러: struct literal에 positional arg 사용 불가

user, err := userModel.Create(email, hashedPassword., role, name)
// → 컴파일 에러: trailing dot
```

## 기대 생성 코드

```go
out, err := auth.HashPassword(auth.HashPasswordRequest{Password: password})
if err != nil { ... }
hashedPassword := out.HashedPassword

user, err := userModel.Create(email, hashedPassword, role, name)
```

## 버그 위치

### 버그 1: `generator/go_target.go` — `buildCallInputFields()` (약 348-356행)

```go
func buildCallInputFields(args []parser.Arg) string {
    var fields []string
    for _, a := range args {
        fields = append(fields, argToCode(a))  // positional만 생성
    }
    return strings.Join(fields, ", ")
}
```

**문제**: `@call` Request struct 초기화에 positional arg (`{ password }`)를 생성함. Go struct literal은 named field (`{Password: password}`)가 필요함.

**수정**: Request struct의 필드명을 매핑하여 `FieldName: value` 형식으로 생성해야 함. Func spec의 Request struct 필드 순서와 @call arg 순서가 1:1 대응하므로, i번째 arg를 i번째 Request 필드명과 매핑.

### 버그 2: `generator/go_target.go` — `argToCode()` (약 319행)

```go
if a.Source != "" {
    return a.Source + "." + a.Field  // Field가 비어있으면 "source." 생성
}
```

**문제**: `@call` 결과 변수(예: `hashedPassword`)가 후속 시퀀스에서 arg로 사용될 때, `a.Field`가 빈 문자열이면 `hashedPassword.`(trailing dot)이 생성됨.

**수정**: `@call` 결과 변수는 이미 값 자체를 담고 있으므로, `a.Field`가 비어있을 때는 `a.Source`만 반환해야 함.

```go
if a.Source != "" {
    if a.Field == "" {
        return a.Source
    }
    return a.Source + "." + a.Field
}
```

### 버그 1 관련: `generator/go_templates.go` — call 템플릿 (약 71, 79행)

현재 템플릿이 `Request{ {{.InputFields}} }` 형식인데, `InputFields`가 named field 형식이 되면 자연히 해결됨.

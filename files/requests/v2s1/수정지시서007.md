# 수정지시서 007: @call args를 JSON 형태 named field로 변경

## 배경

수정지시서 005에서 `callFieldName(arg)`으로 arg에서 Request 필드명을 추론하도록 했으나, `user.ID` → `ID`로 추론되어 실제 필드 `UserID`와 불일치. 추론 자체가 잘못된 방향이다.

`@state`, `@auth`와 동일하게 `{Key: value}` 형태로 필드명을 명시적으로 선언해야 한다.

## 현재 문법 (positional)

```go
// @call string accessToken = auth.IssueToken(user.ID, user.Email, user.Role)
```

생성 코드 (`callFieldName` 추론):
```go
auth.IssueToken(auth.IssueTokenRequest{ID: user.ID, Email: user.Email, Role: user.Role})
// ID가 아니라 UserID여야 함 → 컴파일 에러
```

## 변경 문법 (named field)

```go
// @call string accessToken = auth.IssueToken({UserID: user.ID, Email: user.Email, Role: user.Role})
```

생성 코드:
```go
auth.IssueToken(auth.IssueTokenRequest{UserID: user.ID, Email: user.Email, Role: user.Role})
```

## 수정 범위

### 1. parser/parser.go — `@call` arg 파싱

현재 `@call` args를 positional `(arg1, arg2)` 로 파싱함.
변경: `({Key1: val1, Key2: val2})` 형태를 파싱하여 `Arg.Field`에 명시적 필드명, `Arg.Source`에 값 소스를 저장.

```
// @call string accessToken = auth.IssueToken({UserID: user.ID, Email: user.Email, Role: user.Role})
```

파싱 결과:
```go
Args: []Arg{
    {Field: "UserID", Source: "user", SubField: "ID"},
    {Field: "Email", Source: "user", SubField: "Email"},
    {Field: "Role", Source: "user", SubField: "Role"},
}
```

Guard form (결과 없는 @call)도 동일:
```go
// @call auth.VerifyPassword({PasswordHash: user.PasswordHash, Password: request.Password})
```

### 2. generator/go_target.go — `buildCallInputFields()` 수정

수정지시서 005의 `callFieldName` 추론 로직 제거.
`Arg.Field`를 그대로 사용:

```go
func buildCallInputFields(args []parser.Arg) string {
    var fields []string
    for _, a := range args {
        name := a.Field        // 명시적 필드명 (파서에서 파싱됨)
        value := argToCode(a)
        fields = append(fields, name+": "+value)
    }
    return strings.Join(fields, ", ")
}
```

`callFieldName()` 함수 삭제.

### 3. validator — @call 필드명 검증 추가

`@call` args의 `Field`가 실제 Func spec Request struct 필드명에 존재하는지 검증.
없으면 ERROR:
```
@call auth.IssueToken — Request 필드 "ID"가 IssueTokenRequest에 없습니다
```

### 4. 하위 호환

positional 문법 `(arg1, arg2)`는 더 이상 지원하지 않음. `@call`에는 반드시 `({...})` 형태 사용 필수.
기존 SSaC 파일(dummy-study 등)도 마이그레이션 필요.

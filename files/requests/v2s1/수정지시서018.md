# 수정지시서 018: Go 예약어 파라미터명 검증

## 배경

DDL 컬럼명이 `type`, `range`, `select` 등 Go 예약어인 경우, ssac-model 코드젠이 해당 이름을 그대로 Go 함수 파라미터명으로 사용하여 컴파일 에러가 발생한다.

```
type TransactionModel interface {
	Create(amount int64, gigID int64, type string) (*Transaction, error)
}
```

`type string`은 Go 문법 에러이다. SSaC validate 단계에서 사전에 차단해야 한다.

## 변경 요청

### 1. SSaC validate에서 Go 예약어 파라미터명 ERROR 처리

sqlc 쿼리 파라미터명 또는 DDL 컬럼명이 Go 예약어와 충돌하는 경우, ssac-model 인터페이스 생성 전에 ERROR를 출력한다.

Go 예약어 목록 (25개):
```
break, case, chan, const, continue, default, defer, else, fallthrough,
for, func, go, goto, if, import, interface, map, package, range,
return, select, struct, switch, type, var
```

### 2. ERROR 메시지 형식

```
ERROR: DDL column "type" in table "transactions" is a Go reserved word — rename the column (e.g. "tx_type")
```

### 3. 검증 위치

ssac-model 인터페이스를 생성하는 단계에서, 파라미터명 목록을 순회할 때 Go 예약어 여부를 확인한다.

## 검증 방법

```bash
# transactions 테이블에 "type" 컬럼이 있는 프로젝트에서:
ssac validate <project>
# → ERROR 출력 확인
```

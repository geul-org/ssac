# 수정지시서004: @auth 코드젠 — authz.Claims 제거 + CheckRequest 필드 정리

## 발신

fullend 프로젝트

## 문제

SSaC의 `@auth` 시퀀스 코드젠이 `authz.Claims` 타입을 참조하지만, fullend의 `pkg/authz`에 `Claims` 타입이 존재하지 않는다. 컴파일 에러 발생.

### 현재 SSaC가 생성하는 코드

```go
if _, err = authz.Check(authz.CheckRequest{
    Action: "PublishGig",
    Resource: "gig",
    Claims: authz.Claims{UserID: currentUser.ID},   // ← 존재하지 않는 타입
    ResourceID: gig.ID,
    Role: currentUser.Role,
    UserID: currentUser.ID,                           // ← Claims.UserID와 중복
}); err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "Forbidden"})
    return
}
```

### fullend `pkg/authz`의 실제 CheckRequest

```go
type CheckRequest struct {
    Action     string
    Resource   string
    UserID     int64
    Role       string
    ResourceID int64
}
```

`Claims` 필드 없음. `UserID`는 CheckRequest에 직접 있음.

### 문제점 2가지

1. **`authz.Claims` 타입 미존재** — 컴파일 에러
2. **`UserID` 중복** — `Claims: authz.Claims{UserID: currentUser.ID}` + `UserID: currentUser.ID` 로 같은 값이 두 번

## 원인

`generator/go_helpers.go:171`에서 `currentUser` 참조 감지 시 `ClaimsCode`를 하드코딩:

```go
d.ClaimsCode = "Claims: authz.Claims{UserID: currentUser.ID}, "
```

이 코드는 fullend의 `authz.CheckRequest`에 `Claims` 필드가 있다고 가정하지만, 실제로는 `UserID`가 CheckRequest의 직접 필드다.

## 수정 방안

### `generator/go_helpers.go`

`ClaimsCode` 생성 로직을 제거하고, `currentUser` 참조 시 `UserID`와 `Role`을 `InputFields`에 직접 포함:

```go
// 변경 전
if seq.Type == parser.SeqAuth && hasCurrentUserRef(inputs) {
    inputs = copyInputs(inputs)
    if _, hasRole := inputs["Role"]; !hasRole {
        inputs["Role"] = "currentUser.Role"
    }
    d.ClaimsCode = "Claims: authz.Claims{UserID: currentUser.ID}, "
}

// 변경 후
if seq.Type == parser.SeqAuth && hasCurrentUserRef(inputs) {
    inputs = copyInputs(inputs)
    if _, hasRole := inputs["Role"]; !hasRole {
        inputs["Role"] = "currentUser.Role"
    }
    if _, hasUserID := inputs["UserID"]; !hasUserID {
        inputs["UserID"] = "currentUser.ID"
    }
    // ClaimsCode 제거 — UserID는 CheckRequest의 직접 필드
}
```

### `generator/go_helpers.go` — `seqTemplateData` 구조체

`ClaimsCode` 필드 제거:

```go
// 변경 전
type seqTemplateData struct {
    ...
    ClaimsCode string // "Claims: authz.Claims{UserID: currentUser.ID}, " 또는 ""
    ...
}

// 변경 후
type seqTemplateData struct {
    ...
    // ClaimsCode 필드 삭제
    ...
}
```

### `generator/go_templates.go`

@auth 템플릿에서 `{{.ClaimsCode}}` 제거:

```go
// 변경 전
authz.Check(authz.CheckRequest{Action: "{{.Action}}", Resource: "{{.Resource}}", {{.ClaimsCode}}{{.InputFields}} })

// 변경 후
authz.Check(authz.CheckRequest{Action: "{{.Action}}", Resource: "{{.Resource}}", {{.InputFields}} })
```

### 올바른 생성 결과

```go
if _, err = authz.Check(authz.CheckRequest{
    Action: "PublishGig",
    Resource: "gig",
    ResourceID: gig.ID,
    Role: currentUser.Role,
    UserID: currentUser.ID,
}); err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "Forbidden"})
    return
}
```

## 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_helpers.go` | `ClaimsCode` 필드 제거, `UserID` 직접 주입 |
| `generator/go_templates.go` | `{{.ClaimsCode}}` 제거 |
| `generator/go_handler_test.go` | `Claims: authz.Claims{UserID: currentUser.ID}` assertion → `UserID: currentUser.ID` |

# 수정지시서 031: @response 템플릿에서 HTTP 상태 코드 완전 제거

## 배경

`@response` 성공 응답의 HTTP 상태 코드는 OpenAPI가 SSOT이다. SSaC는 "무엇을 반환하는가"만 선언하며, "HTTP 몇 번인가"는 일절 관여하지 않는다.

현재 `response`/`response_direct` 템플릿이 `http.StatusOK` (200)을 하드코딩하고 있어, OpenAPI에 204나 201로 정의된 엔드포인트도 무조건 200으로 생성된다 (BUG014).

Guard 에러 코드(`@empty` 404, `@state` 409, `@auth` 403 등)는 SSaC 소관이므로 변경하지 않는다.

## 변경 요청

### 1. `generator/go_templates.go` — response 템플릿

`http.StatusOK` → 치환 마커 `__RESPONSE_STATUS__`로 변경. SSaC는 HTTP 코드를 결정하지 않는다. fullend가 OpenAPI를 보고 이 마커를 실제 상태 코드로 치환한다.

```go
// 변경 전
{{- define "response" -}}
	c.JSON(http.StatusOK, gin.H{
		...
	})
{{end}}

{{- define "response_direct" -}}
	c.JSON(http.StatusOK, {{.Target}})
{{end}}

// 변경 후
{{- define "response" -}}
	c.JSON(__RESPONSE_STATUS__, gin.H{
		...
	})
{{end}}

{{- define "response_direct" -}}
	c.JSON(__RESPONSE_STATUS__, {{.Target}})
{{end}}
```

### 2. 기본값 없음

`templateData`에 `StatusCode` 필드를 추가하지 않는다. SSaC는 상태 코드를 전혀 모른다. 고정 문자열 `__RESPONSE_STATUS__`를 템플릿에 직접 쓴다.

### 3. `generator/go_helpers.go` — 변경 없음

response 시퀀스의 templateData 구성에서 상태 코드 관련 로직이 없으므로 변경 불필요.

## 변경 범위

| 파일 | 변경 |
|---|---|
| `generator/go_templates.go` | `response` 템플릿: `http.StatusOK` → `__RESPONSE_STATUS__` |
| `generator/go_templates.go` | `response_direct` 템플릿: `http.StatusOK` → `__RESPONSE_STATUS__` |
| `generator/go_handler_test.go` | `http.StatusOK` assertion → `__RESPONSE_STATUS__` assertion |

## 마커 규약

| 마커 | 의미 | 치환 주체 |
|---|---|---|
| `__RESPONSE_STATUS__` | 성공 응답 HTTP 상태 코드 | fullend gluegen 후처리 |

fullend가 OpenAPI를 보고:
- 200 → `http.StatusOK`
- 201 → `http.StatusCreated`
- 204 → 줄 전체를 `c.Status(http.StatusNoContent)`로 교체
- 미정의 → `http.StatusOK` fallback

## 영향 없는 범위

- `@empty`, `@exists`, `@state`, `@auth`의 `ErrStatus` — 변경 없음 (SSaC 소관)
- `@call`의 `ErrStatus` — 변경 없음
- `@publish`의 `http.StatusInternalServerError` — 변경 없음

## 검증

1. `go test ./...` — 테스트 assertion을 `__RESPONSE_STATUS__`로 업데이트 후 통과
2. 생성 코드에서 `c.JSON(__RESPONSE_STATUS__, ...)` 형태 확인
3. ssac-gen 단독 실행 시 마커가 그대로 출력됨을 확인 (fullend 없이는 컴파일 불가 — 의도된 동작)

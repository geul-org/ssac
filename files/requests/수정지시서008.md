# 수정지시서 008: List 핸들러에서 ParseQueryOpts 직접 생성

## 발신
fullend (BUG005/BUG012 근본 해결)

## 심각도
HIGH — SQL Injection 방지 + 미사용 import 제거

## 배경

현재 SSaC는 List 핸들러에서 `QueryOpts{}`와 수동 파싱 코드(`c.Query("limit")`, `strconv.Atoi` 등)를 생성한다. fullend는 이를 후처리로 `model.ParseQueryOpts(c, cfg)`로 교체하고 수동 파싱을 제거한다.

이 후처리 방식의 문제:
1. 수동 파싱 코드가 allowlist 검증 없이 `opts.SortCol`에 직접 삽입 → SQL Injection (BUG005)
2. 수동 파싱 제거 후 `strconv` import이 남음 → 빌드 실패 (BUG012)
3. 정규식 기반 후처리가 깨지기 쉬움

## 요청

SSaC codegen에서 List 핸들러 생성 시, 수동 파싱 대신 `model.ParseQueryOpts(c, cfg)` 호출 코드를 직접 생성하도록 변경.

## ParseQueryOpts 시그니처

fullend가 generated 코드의 `model/queryopts.go`에 아래 타입과 함수를 제공한다:

```go
package model

type QueryOptsConfig struct {
    Pagination *PaginationConfig
    Sort       *SortConfig
    Filter     *FilterConfig
}

type PaginationConfig struct {
    Style        string // "offset" or "cursor"
    DefaultLimit int
    MaxLimit     int
}

type SortConfig struct {
    Allowed   []string
    Default   string
    Direction string // "asc" or "desc"
}

type FilterConfig struct {
    Allowed []string
}

func ParseQueryOpts(c *gin.Context, cfg QueryOptsConfig) QueryOpts
```

## SSaC가 생성해야 하는 코드

SSaC는 OpenAPI `x-pagination`, `x-sort`, `x-filter` 확장을 이미 읽고 있다. 이 정보로 config 리터럴을 구성:

### 현재 (수동 파싱)
```go
func (h *Handler) ListGigs(c *gin.Context) {
    opts := QueryOpts{}
    if v := c.Query("limit"); v != "" {
        opts.Limit, _ = strconv.Atoi(v)
    }
    if v := c.Query("offset"); v != "" {
        opts.Offset, _ = strconv.Atoi(v)
    }
    allowedSort := map[string]bool{"budget": true, "created_at": true}
    if v := c.Query("sort"); allowedSort[v] {
        opts.SortCol = v
    }
    // ...
}
```

### 변경 후 (ParseQueryOpts)
```go
func (h *Handler) ListGigs(c *gin.Context) {
    opts := model.ParseQueryOpts(c, model.QueryOptsConfig{
        Pagination: &model.PaginationConfig{Style: "offset", DefaultLimit: 20, MaxLimit: 100},
        Sort:       &model.SortConfig{Allowed: []string{"budget", "created_at"}, Default: "created_at", Direction: "desc"},
        Filter:     &model.FilterConfig{Allowed: []string{"status", "budget"}},
    })
    // ...
}
```

## config 매핑 규칙

| OpenAPI 확장 | Config 필드 |
|---|---|
| `x-pagination.style` | `PaginationConfig.Style` |
| `x-pagination.defaultLimit` | `PaginationConfig.DefaultLimit` |
| `x-pagination.maxLimit` | `PaginationConfig.MaxLimit` |
| `x-sort.allowed` | `SortConfig.Allowed` |
| `x-sort.default` | `SortConfig.Default` |
| `x-sort.direction` | `SortConfig.Direction` |
| `x-filter.allowed` | `FilterConfig.Allowed` |

`x-pagination`이 없으면 `Pagination: nil`, 마찬가지로 `x-sort`/`x-filter`도 없으면 nil.

## 효과

- SSaC가 안전한 코드를 직접 생성 → fullend 후처리(`transformSource`, `removeManualQueryParsing`) 불필요
- `strconv` import 문제 자동 해소
- SQL Injection 방지가 생성 시점에서 보장됨

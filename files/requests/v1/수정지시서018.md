# 수정지시서018: validateResponse 역방향 에러에 수정 가이드 추가

## 배경

수정지시서016에서 추가한 역방향 검증(OpenAPI response → SSaC @var)이 에러를 보고할 때, "뭐가 빠졌는지"만 알려주고 "어떻게 고쳐야 하는지"는 안 알려준다. 사용자가 에러를 보고 바로 조치할 수 있도록 수정 가이드를 포함해야 한다.

## 현재

```
ERROR: OpenAPI response에 "instructor" 필드가 있지만 SSaC @var에 없습니다
```

## 수정 후

```
ERROR: OpenAPI response에 "instructor" 필드가 있지만 SSaC @var에 없습니다
  → @sequence response json에 // @var instructor를 추가하고, instructor를 조회하는 시퀀스도 함께 작성하세요
```

## 수정

### `validator/validator.go` — `validateResponse` 함수의 역방향 블록

에러 생성 시 `Message`를 수정:

```go
// 기존
errs = append(errs, ctx.err("@var", fmt.Sprintf("OpenAPI response에 %q 필드가 있지만 SSaC @var에 없습니다", field)))

// 수정
errs = append(errs, ValidationError{
	FileName: ctx.fileName,
	FuncName: ctx.funcName,
	SeqIndex: ctx.seqIndex,
	Tag:      "@var",
	Message:  fmt.Sprintf("OpenAPI response에 %q 필드가 있지만 SSaC @var에 없습니다\n  → @sequence response json에 // @var %s를 추가하고, %s를 조회하는 시퀀스도 함께 작성하세요", field, field, field),
})
```

## 영향 파일

| 파일 | 변경 |
|------|------|
| `validator/validator.go` | `validateResponse` 역방향 에러 메시지에 수정 가이드 추가 |

# 수정지시서 009 실행결과

## 1. guard state @message — 이미 동작

템플릿이 `{{.Message}}`를 사용하고, `defaultMessage`가 기본값을 제공하므로 변경 없이 동작.

- 기본값: `"상태 전이가 허용되지 않습니다"`
- 커스텀: `@message "현재 상태에서 공개할 수 없습니다"` → 그대로 출력

## 2. password @message — 이미 동작

guard state과 동일. 템플릿이 `{{.Message}}`를 사용.

- 기본값: `"비밀번호가 일치하지 않습니다"`
- 커스텀: `@message "비밀번호가 틀렸습니다"` → 그대로 출력

## 3. authorize @message — 수정 완료

기존 문제: authorize 템플릿에서 Forbidden 메시지가 `"권한이 없습니다"`로 하드코딩. `@message`가 내부 에러(InternalServerError)에만 적용됨.

수정:
- authorize 템플릿: Forbidden에 `{{.Message}}` 사용, 내부 에러는 `"권한 확인 실패"` 하드코딩
- `defaultMessage(SeqAuthorize)`: `"권한 확인 실패"` → `"권한이 없습니다"` (Forbidden용 기본값)

```go
// @message 없음 → 기본값
http.Error(w, "권한 확인 실패", http.StatusInternalServerError)  // 내부 에러 (고정)
http.Error(w, "권한이 없습니다", http.StatusForbidden)           // 비즈니스 거부 (기본)

// @message "커스텀 메시지"
http.Error(w, "권한 확인 실패", http.StatusInternalServerError)  // 내부 에러 (고정)
http.Error(w, "커스텀 메시지", http.StatusForbidden)             // 비즈니스 거부 (커스텀)
```

# 수정지시서 015: QueryOpts에서 Includes 필드 제거

## 배경

`x-include`는 코드젠 시 관련 테이블 JOIN을 항상 적용하기 위한 메타데이터이다.
런타임에 클라이언트가 `?include=` 쿼리 파라미터로 선택하는 기능이 아니다.

fullend 쪽에서는 이미 `?include=` 파싱과 조건부 include 로직을 제거했다.
SSaC에서도 `QueryOpts`의 `Includes` 필드를 제거하여 정합성을 맞춘다.

## 수정 항목

### 1. `generator/go_target.go` — QueryOpts struct에서 Includes 제거

**현재** (line 1075):
```go
Includes []string
```

**수정**: 해당 라인 삭제.

QueryOpts struct 최종 형태:
```go
type QueryOpts struct {
    Limit   int
    Offset  int
    Cursor  string
    SortCol string
    SortDir string
    Filters map[string]string
}
```

## 영향 범위

- `Includes` 필드를 참조하는 코드가 SSaC 내부에 없으므로 (`parseIncludes` 헬퍼도 없음) 삭제만 하면 된다
- fullend 쪽 model_impl은 `opts.Includes`를 더 이상 참조하지 않으므로 호환성 문제 없음

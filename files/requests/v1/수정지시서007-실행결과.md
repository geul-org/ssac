# 수정지시서 007 실행결과

## 1. parser: 재귀 디렉토리 탐색 — 완료

`ParseDir`을 `os.ReadDir` → `filepath.WalkDir`로 변경. import에 `io/fs` 추가, `os` 제거.

하위 디렉토리의 `.go` 파일도 파싱하며, 상대 경로의 첫 번째 디렉토리를 `Domain`으로 파생:
- `service/login.go` → `Domain=""`
- `service/course/create_course.go` → `Domain="course"`

## 2. ServiceFunc에 Domain 필드 추가 — 완료

`parser/types.go`의 `ServiceFunc`에 `Domain string` 필드 추가.

```go
type ServiceFunc struct {
    Name      string
    FileName  string
    Domain    string     // 도메인 폴더명 (e.g. "auth"). 빈 문자열이면 루트.
    Sequences []Sequence
}
```

## 3. generator: 도메인별 서브디렉토리 출력 — 완료

`GenerateWith`에서 `sf.Domain`이 있으면 `outDir/domain/` 서브디렉토리에 출력:

```go
outPath := outDir
if sf.Domain != "" {
    outPath = filepath.Join(outDir, sf.Domain)
    os.MkdirAll(outPath, 0755)
}
path := filepath.Join(outPath, outName)
```

검증:
- flat: `Domain=""` → `outDir/login.go`
- 폴더: `Domain="course"` → `outDir/course/create_course.go`

## 4. generator: 도메인별 package 이름 — 완료

`GenerateFunc`에서 `package service` 하드코딩 → Domain 기반:

```go
pkgName := "service"
if sf.Domain != "" {
    pkgName = sf.Domain
}
```

검증:
- flat: `package service`
- 폴더: `package course`

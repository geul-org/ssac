# 수정지시서 010: request struct json tag를 OpenAPI 필드명 그대로 유지

## 요청자
fullend

## 요약

`generator/go_target.go`의 request struct 생성 시, json tag를 `toSnakeCase()`로 변환하지 말고 **OpenAPI schema의 원본 필드명을 그대로** 사용해야 한다.

## 현상

OpenAPI에서 PascalCase 필드명을 정의:
```yaml
RegisterRequest:
  properties:
    Email:
      type: string
    Password:
      type: string
    Name:
      type: string
```

SSaC에서도 PascalCase로 참조:
```go
// @param Email request
// @param Password request
// @param Name request
```

그러나 생성된 코드의 json tag가 snake_case:
```go
var req struct {
    Email    string `json:"email"`      // ✗ "email"
    Password string `json:"password"`   // ✗ "password"
    Name     string `json:"name"`       // ✗ "name"
}
```

클라이언트가 OpenAPI 스펙대로 `{"Email":"...", "Password":"..."}` 보내면 디코딩 실패 (400 Bad Request).

## 원인

`generator/go_target.go:363`:
```go
jsonTag := toSnakeCase(rp.name)
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", rp.name, rp.goType, jsonTag))
```

`toSnakeCase("Email")` → `"email"`, `toSnakeCase("CourseID")` → `"course_id"` 등으로 변환됨.

## 수정 방법

json tag에 `toSnakeCase()` 대신 **원본 필드명(`rp.name`)을 그대로** 사용:

```go
// 수정 전
jsonTag := toSnakeCase(rp.name)
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", rp.name, rp.goType, jsonTag))

// 수정 후
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", rp.name, rp.goType, rp.name))
```

## 기대 결과

```go
var req struct {
    Email    string `json:"Email"`      // ✓ OpenAPI 일치
    Password string `json:"Password"`   // ✓ OpenAPI 일치
    Name     string `json:"Name"`       // ✓ OpenAPI 일치
}
```

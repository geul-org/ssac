# 수정지시서 007: 도메인 폴더 구조 지원

## 배경

현재 ssac은 `specs/service/*.go`를 flat하게 탐색한다. 서비스가 수백~수천 개로 늘어나면 flat 구조는 관리 불가능하다.

도메인별 폴더로 서비스를 분류할 수 있어야 한다:

```
specs/service/
├── course/
│   ├── create_course.go
│   ├── list_courses.go
│   ├── get_course.go
│   ├── update_course.go
│   └── delete_course.go
├── lesson/
│   ├── create_lesson.go
│   ├── list_lessons.go
│   ├── update_lesson.go
│   └── delete_lesson.go
├── auth/
│   ├── login.go
│   └── register.go
├── enrollment/
│   └── enroll_course.go
├── review/
│   ├── create_review.go
│   ├── list_reviews.go
│   └── delete_review.go
└── payment/
    ├── list_my_payments.go
    └── list_my_enrollments.go
```

## 1. parser: 재귀 디렉토리 탐색

### 현재

```go
// parser.go
func ParseDir(dir string) ([]ServiceFunc, error) {
    entries, err := os.ReadDir(dir)
    for _, entry := range entries {
        if entry.IsDir() || !strings.HasSuffix(entry.Name(), ".go") {
            continue  // ← 하위 디렉토리 무시
        }
```

### 수정

`os.ReadDir` → `filepath.WalkDir`로 변경하여 하위 디렉토리의 `.go` 파일도 파싱한다.

```go
func ParseDir(dir string) ([]ServiceFunc, error) {
    var funcs []ServiceFunc
    err := filepath.WalkDir(dir, func(path string, d fs.DirEntry, err error) error {
        if err != nil || d.IsDir() || !strings.HasSuffix(d.Name(), ".go") {
            return err
        }
        sf, err := ParseFile(path)
        if err != nil {
            return fmt.Errorf("%s 파싱 실패: %w", path, err)
        }
        if sf != nil {
            // 도메인은 service/ 기준 상대 경로의 첫 번째 디렉토리
            rel, _ := filepath.Rel(dir, path)
            if parts := strings.Split(filepath.Dir(rel), string(filepath.Separator)); parts[0] != "." {
                sf.Domain = parts[0]
            }
            funcs = append(funcs, *sf)
        }
        return nil
    })
    return funcs, err
}
```

### 하위 호환

- `specs/service/login.go` (flat) → `Domain = ""` (빈 문자열)
- `specs/service/auth/login.go` (폴더) → `Domain = "auth"`
- 기존 flat 구조 프로젝트는 변경 없이 동작

### 영향 파일

- `parser/parser.go` — `ParseDir` 수정
- `parser/types.go` — `ServiceFunc`에 `Domain string` 필드 추가

## 2. ServiceFunc에 Domain 필드 추가

### 현재

```go
type ServiceFunc struct {
    Name      string
    FileName  string
    Sequences []Sequence
    // ...
}
```

### 수정

```go
type ServiceFunc struct {
    Name      string
    FileName  string
    Domain    string      // 도메인 폴더명 (e.g. "course", "auth"). 빈 문자열이면 루트.
    Sequences []Sequence
    // ...
}
```

### 영향 파일

- `parser/types.go` — `Domain` 필드 추가

## 3. generator: 도메인별 서브디렉토리 출력

### 현재

```go
// generator.go — GenerateWith
path := filepath.Join(outDir, outName)
```

모든 서비스 파일이 `outDir/` 에 flat하게 생성된다.

### 수정

`Domain`이 있으면 서브디렉토리에 출력한다:

```go
// generator.go — GenerateWith
outPath := outDir
if sf.Domain != "" {
    outPath = filepath.Join(outDir, sf.Domain)
    os.MkdirAll(outPath, 0755)
}
path := filepath.Join(outPath, outName)
```

생성 결과:
```
internal/service/
├── course/
│   ├── create_course.go    ← package course
│   ├── list_courses.go
│   └── ...
├── auth/
│   ├── login.go            ← package auth
│   └── register.go
└── ...
```

### package 이름

도메인 폴더에 출력할 때 `package service` 대신 `package {domain}`을 사용한다:

```go
// go_target.go — GenerateFunc
pkgName := "service"
if sf.Domain != "" {
    pkgName = sf.Domain
}
buf.WriteString(fmt.Sprintf("package %s\n\n", pkgName))
```

### 하위 호환

- `Domain = ""` → 기존처럼 `outDir/`에 `package service`로 출력
- `Domain = "course"` → `outDir/course/`에 `package course`로 출력

### 영향 파일

- `generator/generator.go` — `GenerateWith` 출력 경로 분기
- `generator/go_target.go` — `GenerateFunc` package 이름 분기

## 4. generator: 모델 인터페이스도 도메인 분리

### 현재

`GenerateModelInterfaces`가 모든 모델을 `model/models_gen.go` 하나에 생성한다.

### 수정

도메인이 있으면 도메인별로 모델 인터페이스를 분리할 수 있지만, 모델은 도메인 간 공유될 수 있으므로 (e.g. `UserModel`을 auth, enrollment 둘 다 사용) **모델 인터페이스는 단일 `model/` 패키지를 유지한다.**

변경 없음.

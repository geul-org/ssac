# 수정지시서 002: oapi-codegen ServerInterface 시그니처 호환

## 배경

fullend는 oapi-codegen (Apache-2.0)으로 OpenAPI에서 Go `ServerInterface`를 생성한다.
SSaC이 생성하는 서비스 함수가 이 인터페이스를 직접 구현하면 어댑터 없이 라우터에 연결된다.

## 현재 문제

oapi-codegen이 생성하는 `ServerInterface`:

```go
type ServerInterface interface {
    ListCourses(w http.ResponseWriter, r *http.Request)
    GetCourse(w http.ResponseWriter, r *http.Request, courseID int64)
    DeleteCourse(w http.ResponseWriter, r *http.Request, courseID int64)
    CreateLesson(w http.ResponseWriter, r *http.Request, courseID int64)
    DeleteReview(w http.ResponseWriter, r *http.Request, reviewID int64)
    Login(w http.ResponseWriter, r *http.Request)
}
```

SSaC이 현재 생성하는 함수:

```go
func GetCourse(w http.ResponseWriter, r *http.Request) {
    courseID := r.FormValue("CourseID")  // ← path param을 FormValue로 읽음
    ...
}
```

**불일치:**
- path parameter가 있는 엔드포인트에서 시그니처가 다름
- `GetCourse(w, r)` vs `GetCourse(w, r, courseID int64)`
- path parameter 없는 함수(`Login`, `ListCourses`)는 이미 일치

## 요구사항

### 1. OpenAPI path parameter를 함수 인자로 생성

OpenAPI에서 `parameters[].in: path`인 파라미터가 있으면, 해당 함수의 시그니처에 인자를 추가한다.

```go
// 현재
func GetCourse(w http.ResponseWriter, r *http.Request) {
    courseID := r.FormValue("CourseID")
    ...
}

// 변경
func GetCourse(w http.ResponseWriter, r *http.Request, courseID int64) {
    // courseID는 이미 인자로 받았으므로 FormValue 호출 불필요
    ...
}
```

### 2. 인자 순서

oapi-codegen의 규칙: `(w, r)` 뒤에 path parameter를 OpenAPI 정의 순서대로 추가.

```yaml
/courses/{CourseID}/lessons/{LessonID}:
  parameters:
    - name: CourseID
    - name: LessonID
```

→ `func UpdateLesson(w, r, courseID int64, lessonID int64)`

### 3. 인자 타입

OpenAPI schema에서 파생:

| OpenAPI type + format | Go 타입 |
|---|---|
| `integer` + `int64` | `int64` |
| `integer` | `int` |
| `string` | `string` |
| `string` + `uuid` | `string` |

### 4. @param request에서 path parameter 제외

path parameter로 이미 인자에 포함된 변수는 `r.FormValue()` 호출을 생성하지 않는다.

현재 SSaC spec:
```go
// @param CourseID request
```

`CourseID`가 OpenAPI path parameter에 있으면 → 함수 인자에서 받음, `r.FormValue("CourseID")` 생성 안 함.
`CourseID`가 path parameter에 없으면 → 기존대로 `r.FormValue("CourseID")` 생성.

### 5. 인자 이름 변환

oapi-codegen은 PascalCase parameter를 camelCase 인자로 변환한다:
- `CourseID` → `courseID`
- `LessonID` → `lessonID`
- `ReviewID` → `reviewID`

SSaC도 동일한 변환을 적용해야 한다.

## 구현 위치

- `generator/` — 코드젠 시 OpenAPI path parameter 정보 참조
  - SymbolTable의 Operations에 이미 path parameter 정보가 있는지 확인
  - 없다면 generator가 OpenAPI spec을 직접 로드하거나, SymbolTable에 path parameter 정보 추가 필요

## 하위 호환

- OpenAPI spec이 없으면 기존대로 `func Name(w, r)` 시그니처 유지
- path parameter가 없는 함수는 변경 없음

## 검증

```go
// ServerInterface를 직접 구현하는 struct
type Server struct{}

func (s *Server) GetCourse(w http.ResponseWriter, r *http.Request, courseID int64) { ... }
func (s *Server) Login(w http.ResponseWriter, r *http.Request) { ... }

// 컴파일 타임 체크
var _ api.ServerInterface = (*Server)(nil)
```

위 코드가 컴파일되면 시그니처 호환 완료.

# 수정지시서 003 실행결과

## 1. Model 인터페이스 DDL 타입 반영 — 완료

`resolveParamType()`에 3단계 조회 추가:
1. 해당 모델 테이블에서 직접 조회 (`Course` → `courses`)
2. `{Model}ID` 패턴 인식 (`CourseID` → `courses.id` → `int64`)
3. 전체 테이블 순회 (기존 fallback)

`toSnakeCase()` 연속 대문자 처리 수정: `RoomID` → `room_id` (기존: `room_i_d`)

## 2. JSON body 파싱 — 완료

path param 제외 후 request 파라미터 2개 이상 + SymbolTable 있으면 JSON body struct decode 생성.

```go
var req struct {
    Title       string `json:"title"`
    Description string `json:"description"`
    Price       int64  `json:"price"`
}
if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
    http.Error(w, "invalid request body", http.StatusBadRequest)
    return
}
title := req.Title
```

SymbolTable 없으면 기존 `r.FormValue()` 유지 (하위 호환).

## 3. authorize @id 예약어 — 완료

`@id new` → `nil` 변환. 리소스 생성 전 authorize에서 ID가 없음을 표현.

## 4. 출력 폴더 구조 — 스킵

지시서 권장대로 fullend 측에서 outDir을 세분화하여 해결.

51개 전체 통과 (parser 14 + validator 30 + generator 7).

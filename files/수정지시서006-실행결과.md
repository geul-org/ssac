# 수정지시서 006 실행결과

## 1. FindByID에 opts 잘못 전달 — 완료

opts 추가 조건을 함수 레벨 `HasQueryOpts()` → 모델 메서드 레벨 `isListMethod()` 기준으로 변경.

`isListMethod()` 신규: 모델 메서드명이 `List`로 시작하는지 확인.
`hasAnyQueryOpts()` 신규: 심볼 테이블에 QueryOpts를 가진 operation이 있는지 확인.

`opts := QueryOpts{}` 생성도 동일 조건: 시퀀스 중 List 메서드가 있을 때만 생성.

검증:
```go
// get_reservation.go — FindByID에 opts 없음 ✓
reservation, err := reservationModel.FindByID(reservationID)

// list_my_reservations.go — ListByUserID에 opts 있음 ✓
reservations, total, err := reservationModel.ListByUserID(currentUser.UserID, opts)
```

## 2. ListLessons에 opts 누락 — 완료

이슈 1과 동일 근본 원인. `isListMethod()` 기준으로 통합 해결. List 메서드면 항상 opts 전달.

## 3. 리터럴 파라미터 이름 잘못됨 — 완료

3가지 수정:

1. **DDLTable에 ColumnOrder 추가**: `validator/symbol.go`의 `DDLTable`에 `ColumnOrder []string` 필드 추가. `parseDDLTables()`에서 컬럼 파싱 순서 보존.

2. **usedColumns 구축 개선**:
   - dot notation: `enrollment.ID` → `id` + `enrollment_id` (복합 컬럼명 추가)
   - 변수 참조: `splitCamelWords()`로 camelCase 분리 후 DDL 컬럼과 부분 매칭으로 제외 (예: `hashedPassword` → `["hashed", "Password"]` → `password_hash` 제외)

3. **resolveLiteralParamName**: 알파벳순 → DDL 정의 순서(`ColumnOrder`)로 첫 번째 미사용 string 컬럼 선택.

검증: `UpdateStatus(reservationID int64, status string)` — `"cancelled"` 리터럴이 `status`로 매핑됨.

## 4. total이 response에 미포함 — 완료

`GenerateFunc()`에서 `funcHasTotal` 플래그를 추적하여 response 시퀀스에 전달. response json 템플릿에 `"total": total` 조건부 추가.

```go
// list_my_reservations.go 생성 결과
json.NewEncoder(w).Encode(map[string]interface{}{
    "reservations": reservations,
    "total":        total,
})
```

## 5. sqlc 쿼리 이름 접두사 — 완료

`stripModelPrefix()` 신규: `"CourseFindByID"` + `"Course"` → `"FindByID"`. 접두사 없는 기존 쿼리도 그대로 동작.

`loadSqlcQueries()`에서 `-- name:` 파싱 시 `stripModelPrefix()` 적용.

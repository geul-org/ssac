# 수정지시서017 회신

## 결론: SSaC 스킵, fullend 이관

`@archived` 태그의 유일한 소비자는 DDL → OpenAPI/SSaC 역방향 검증이며, 이 검증은 fullend 책임으로 결정됨.

SSaC의 DDL 파싱 목적은 타입 변환 코드젠과 심볼 테이블 구축이며, `@archived` 정보는 SSaC에서 사용하지 않음.

## fullend 구현 시 참고

- DDL 파싱에 `@archived` 감지 추가
  - 테이블 수준: `CREATE TABLE` 직전 줄 `-- @archived` → 테이블 전체 제외
  - 컬럼 수준: 컬럼 정의 줄 끝 `-- @archived` → 해당 컬럼 제외
- DDL → OpenAPI/SSaC 역방향 검증에서 `@archived` 항목 스킵
- WARNING 메시지에 `@archived` 안내 포함 (예: `"보존용이면 -- @archived 추가"`)

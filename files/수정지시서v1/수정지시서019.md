# 수정지시서019: @param source가 변수 참조일 때 codegen 오류

## 현상

```go
// @sequence get
// @model Course.FindByID
// @param CourseID request
// @result course Course
//
// @sequence get
// @model User.FindByID
// @param InstructorID course    ← source가 이전 @result 변수
// @result instructor User
```

위 SSaC를 codegen하면:

```go
// 생성된 코드 (잘못됨)
instructor, err := h.UserModel.FindByID(instructorID)
```

### 문제 1: @param source 변수 참조 미해결

`@param InstructorID course`에서 source `course`는 이전 시퀀스의 `@result course Course`를 가리킨다.
생성 코드는 `course.InstructorID`여야 하는데 `instructorID`(미정의 변수)로 생성됨.

**기대 코드:**
```go
instructor, err := h.UserModel.FindByID(course.InstructorID, opts)
```

### 문제 2: FindByID에 QueryOpts 누락

`FindByID(int64, model.QueryOpts)` 시그니처인데, 생성된 코드에서 `opts` 인자가 빠짐.

## 재현

```bash
cd ~/.clari/repos/fullend
./fullend gen specs/dummy-lesson artifacts/dummy-lesson
cd artifacts/dummy-lesson/backend
go mod tidy && go build ./...
# → internal/service/course/get_course.go:43: undefined: instructorID
# → not enough arguments in call to h.UserModel.FindByID
```

## 요청

1. `@param <Field> <varName>` — varName이 `request`가 아닌 이전 @result 변수일 때 `varName.Field`로 codegen
2. 모든 model 메서드 호출에 `QueryOpts` 인자 포함

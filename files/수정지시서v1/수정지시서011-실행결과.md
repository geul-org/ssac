## 2. `@func` 패키지 분리 코드젠 — 완료

### 파싱
- `parser/types.go`: `Sequence` struct에 `Package string` 필드 추가
- `parser/parser.go`: `@func auth.verifyPassword` → `Package="auth"`, `Func="verifyPassword"` 분리

### 검증
- `validator/validator.go`: `@func`에 `.`이 없으면 ERROR (`package.funcName 형식 필수`)
- `validateCallTarget`: `@func` with package → 외부 패키지이므로 model/ 교차검증 스킵

### 코드젠
- `generator/go_templates.go`: `call_func` 템플릿 변경
  ```
  out, err := auth.VerifyPassword(auth.VerifyPasswordInput{PasswordHash: user.PasswordHash, Password: password})
  ```
- `generator/go_target.go`:
  - `templateData`에 `PkgName`, `FuncMethod`, `InputFields`, `ResultField`, `FuncErrStatus` 추가
  - `buildInputFields()` 신규 — @param을 Input struct 필드 문자열로 변환
  - `ucFirst()` 신규 — 첫 글자 대문자 변환
  - `@result` 없는 call: guard형 → `http.StatusUnauthorized`
  - `@result` 있는 call: value형 → `http.StatusInternalServerError`
  - import에 패키지명 추가

### 생성 코드 예시

```go
// guard형 (@result 없음)
_, err = auth.VerifyPassword(auth.VerifyPasswordInput{PasswordHash: user.PasswordHash, Password: password})
if err != nil {
    http.Error(w, "비밀번호가 일치하지 않습니다", http.StatusUnauthorized)
    return
}

// value형 (@result 있음)
out, err := billing.CalculateRefund(billing.CalculateRefundInput{Reservation: reservation})
if err != nil {
    http.Error(w, "calculateRefund 호출 실패", http.StatusInternalServerError)
    return
}
refund := out.Refund
```

# 수정지시서 009: guard state, password, authorize에 @message 지원 추가

## 요청자
fullend

## 요약

`guard state`, `password`, `authorize` 시퀀스에 `@message` 선택 태그를 지원한다. `guard nil`/`guard exists`와 동일한 방식. `@message`가 없으면 기존 기본값 유지.

## 대상 3개 타입

### 1. guard state

현재:
```go
// @sequence guard state course
// @param course.Published
```
→ `http.Error(w, "상태 전이가 허용되지 않습니다", http.StatusConflict)`

변경 후:
```go
// @sequence guard state course
// @param course.Published
// @message "이 강의는 현재 상태에서 공개할 수 없습니다"
```
→ `http.Error(w, "이 강의는 현재 상태에서 공개할 수 없습니다", http.StatusConflict)`

기본값: `"상태 전이가 허용되지 않습니다"`

### 2. password

현재:
```go
// @sequence password
// @param user.PasswordHash
// @param Password request
```
→ `http.Error(w, "비밀번호가 일치하지 않습니다", http.StatusUnauthorized)`

변경 후:
```go
// @sequence password
// @param user.PasswordHash
// @param Password request
// @message "비밀번호가 틀렸습니다. 다시 시도해주세요"
```
→ `http.Error(w, "비밀번호가 틀렸습니다. 다시 시도해주세요", http.StatusUnauthorized)`

기본값: `"비밀번호가 일치하지 않습니다"`

### 3. authorize

현재:
```go
// @sequence authorize
// @action update
// @resource course
// @id CourseID
```
→ `http.Error(w, "권한이 없습니다", http.StatusForbidden)`

변경 후:
```go
// @sequence authorize
// @action update
// @resource course
// @id CourseID
// @message "이 강의를 수정할 권한이 없습니다. 강의 소유자에게 문의하세요"
```
→ `http.Error(w, "이 강의를 수정할 권한이 없습니다. 강의 소유자에게 문의하세요", http.StatusForbidden)`

기본값: `"권한이 없습니다"`

참고: authorize의 내부 에러 메시지(`"권한 확인 실패"`, StatusInternalServerError)는 @message 대상이 아니다. 비즈니스 거부 메시지(StatusForbidden)만 커스텀 대상.

## 수정 범위

### parser

변경 없음. `@message`는 이미 모든 시퀀스에서 파싱 가능한 공용 태그.

### validator

변경 없음. `@message`는 선택 태그이므로 검증 규칙 추가 불필요.

### generator

3개 타입의 템플릿에서 `Sequence.Message`가 비어있지 않으면 해당 값 사용, 비어있으면 기존 기본값 사용. `guard nil`/`guard exists`와 동일한 패턴.

## 테스트

- `TestGenerateGuardStateWithMessage`: 커스텀 메시지 출력 확인
- `TestGeneratePasswordWithMessage`: 커스텀 메시지 출력 확인
- `TestGenerateAuthorizeWithMessage`: Forbidden 메시지 커스텀 확인
- 기존 테스트: `@message` 없을 때 기본값 유지 확인
